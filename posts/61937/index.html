点云物体检测

发表于2023-11-29|更新于2023-12-27|点云图像

0 基本方法思路

点云方面的目标检测主要有四种：

  • 通过点云多个角度的投影进行目标检测
  • 将点云通过pointnet构建成类似图片形式的feature map
    • 也就是通过pointnet将点云目标检测转化为图像目标检测，并使用二维的CNN进行处理
    • 也分为Two-stage和One-stage方法
  • 直接在点的层面进行分析
    • 将二维的CNN转化为点的操作，如pointnet
    • 同样分为Two-stage和One-stage
  • 点云和图像融合

1 MV3D

通过一个3D proposal可以将其投影到不同的维度，在这里可以投影到鸟瞰图、正视图和图片上，然后分别进行ROI Pooling，并最终进行一个分类和回归。现在用的人不多了。

2 VoxelNet

2.1 总体思路

这个方法，简单来说就是利用了Voxel Grid的思想，首先将空间分为很多个小格子，无规律且稀疏的点云数据就变为了三维数据，这个时候就可以通过3D的CNN来进行操作了，变为2D的feature map之后，就可以进行RPN，也就是和图像方法一样了。

举例来说，如上图所示我们先体素化一下，将空间分为10×400×35210\times400\times35210×400×352个体素格。之后我们认为，每一个格子有TTT个点，如果多了就随机down-sampling，如果少了就重复点，经过操作每个格子有相同的点数量后方便GPU进行并行操作。

之后就经过他所设计的VFE结构，说白了就是两个PointNet合到一起罢了，最终每一个voxel都生成了一个一维长度为128的特征向量（经过Max Poolling），也就是最终结果为128×10×400×352128\times10\times400\times352128×10×400×352

但是，这么多格子，要每一个都跑一遍所谓的VFE吗？很明显不是的，因为90%的格子都是空的。所以我们进行如下操作：

  • 我们定义KKK为（最多）有多少个格子是存在点的
  • 定义TTT为每个格子（最多）有多少个点
  • 之后我们搞两个容器：
    • 一个容器装所有的点，也就是K×T×7K\times T \times 7K×T×7。为什么是7？因为每个点的信息包括xi,yi,zi,ri,xi−vix,yi−viy,zi−zizx_i, y_i, z_i, r_i, x_i - v_{ix}, y_i - v_{iy}, z_i - z_{iz}xi​,yi​,zi​,ri​,xi​−vix​,yi​−viy​,zi​−ziz​，就是坐标、反射率以及在每个格子的相对位置。
    • 一个容器装每一个格子的信息，也就是K×3K \times 3K×3，就是每一个格子的中心点的坐标
  • 遍历每一个点，得到其所对应的voxel（通过hash表就可以获得），然后填入两个容器中
  • 最终结果就只有K×T×128K \times T \times 128K×T×128
  • 再通过两个容器重新映射回原来的坐标系，变成128×100×400×352128\times 100 \times 400\times 352128×100×400×352

之后我们再让这个东西经过几层3D卷积层，具体来说参数如下：

  • Output channel # 64, kernel (3, 3, 3), stride (2, 1, 1), padding (1, 1, 1)
  • Output channel # 64, kernel (3, 3, 3), stride (1, 1, 1), padding (0, 1, 1)
  • Output channel # 64, kernel (3, 3, 3), stride (2, 1, 1), padding (1, 1, 1)

最终输出的张量大小变为64×2×400×35264 \times 2 \times 400 \times 35264×2×400×352，然后给它把第二维压了，直接reshape成128×400×352128 \times 400 \times 352128×400×352

这就是一个类似图片的feature map了！（第一维是通道数）

经过RPN网络，每个Anchor Box都有0°和90°两种，即k=2k = 2k=2。其他的就和普通的RPN没啥区别。

分类说完了，回归怎么办呢？

2.3 数据增广 (Data Argumentation)

2.3.1 Global

2.3.1.1 Global translation

这个是没有必要的，毕竟每一个点都针对自己所属的voxel进行了归一化的操作。

2.3.1.2 Global rotation

这个可以有，可以提高模型的泛用性。但是旋转角度不宜过高，一般为θ∈[−π4,π4]\theta \in [-\frac{\pi}{4}, \frac{\pi}{4}]\\θ∈[−4π​,4π​]

为什么要限定呢？因为比如上面这张图就是一个道路的点云数据，如果旋转过大，那可能车就会向着一棵树或者一个路边商铺行驶，就会很奇怪。

2.3.2 Ground truth

意思就是可以把这个点云物体抠出来，旋转平移放到其他的地方，可以提高数据量和训练效果。

  • 对于旋转而言，需要保证θ∈[−π10,π10]\theta \in [-\frac{\pi}{10}, \frac{\pi}{10}]\\θ∈[−10π​,10π​]
  • 对于平移而言，需要保证[Δx,Δy,Δz],Δ∗∼N(0,1)[\Delta x, \Delta y, \Delta z], \Delta *\sim N(0, 1)[Δx,Δy,Δz],Δ∗∼N(0,1)
  • 如果最终移动的物体与其他的ground truth有交集，那么就放弃增广

也很好理解，毕竟点云有一些特征，比如距离雷达近的物体点的数量多，远的物体点的数量少；又比如扫描到的点应该是面向雷达的轮廓，应该符合实际；等等等等。

3 Pillar

3.1 总体思路

和上面不同的是，直接把voxel变为长条状的，顶天立地，然后通过PointNet直接就变成了C×H×WC \times H \times WC×H×W的feature map，然后直接跑SSD之类的就可以了

3.2 Loss Function

L=1Npos(βlocLloc+βclsLcls+βdirLdir)L = \frac{1}{N_{pos}}(\beta_{loc}L_{loc} + \beta_{cls}L_{cls} + \beta_{dir}L_{dir})\\L=Npos​1​(βloc​Lloc​+βcls​Lcls​+βdir​Ldir​)

这个就是和之前的也很像，loc是回归的损失函数，不同之处在于角度方面的处理，不同于以往的Δθ=θg−θa\Delta \theta = \theta^g - \theta^aΔθ=θg−θa，它采用了Δθ=sin⁡(θg−θa)\Delta \theta = \sin(\theta^g - \theta^a)Δθ=sin(θg−θa)，是为了避免0°和360°方向是一样的，但是数值却差了很多的问题。但是这又带来了新的问题：每隔180°方向就会一样怎么办？

因此我们引入了dir参数，也就是方向参数。这个可以自己定，可以以90°为一个分隔来进行赋值。

不一样的还有classification部分，不同于以前的分为pos和neg并有不同的权值，它只用了一个部分就完成了，这是因为他用到了FocalLoss

不同的交叉熵（Cross Entropy Loss）是LCE(pt)=−log⁡ptL_{CE}(p_t) = -\log p_tLCE​(pt​)=−logpt​

而Focal Loss为LFL=−(1−pt)γlog⁡ptL_{FL} = -(1 - p_t)^\gamma\log p_tLFL​=−(1−pt​)γlogpt​

有什么好处呢？比如说我们现在有一个ground truth "human"其他的都是background。那么比如我们的网络预测了100个boxes：

  • 对于前99个boxes而言，有pbackground=0.6p_{background} = 0.6pbackground​=0.6
  • 对于第100个boxed而言，有phuman=0.2p_{human} = 0.2phuman​=0.2

那么交叉熵得到的结果为LCE=∑t−log⁡pt=−99log⁡0.6−log⁡0.2=50.57+1.61L_{CE} = \sum_t-\log p_t = -99 \log 0.6 - \log 0.2 = 50.57 + 1.61LCE​=∑t​−logpt​=−99log0.6−log0.2=50.57+1.61

可以发现ground truth所造成的影响微乎其微，而在目标检测中，绝大多数的box都是background，这就很容易让网络认为所有的东西都是background

而如果是FocalLoss，就会有LFL=−99(1−0.6)2log⁡0.6−(1−0.2)2log⁡0.2=8.09+1.03L_{FL} = -99(1 - 0.6) ^ 2 \log 0.6 - (1 - 0.2) ^ 2 \log 0.2 = 8.09 + 1.03LFL​=−99(1−0.6)2log0.6−(1−0.2)2log0.2=8.09+1.03

可以看到这个ground truth就造成了很大的变化。

4 PointRCNN

上面就是PointRCNN的结构图。它主要分为三个大块：PointNet++、RPN和ROI pooling。

4.1 PointNet++

这一块没啥说的，实际上就是对每一个点进行处理，得到每个点的feature map。

4.2 RPN

得到feature map之后，用这些数据进行分类和回归。

4.2.1 分类

分类是指：分类每一个点是foreground还是background。在这个模型中，前景点自然就是我们认为是我们所需要目标识别出来的东西所对应的点，其他点都是背景点。

自然，这个问题中背景点是远远多于前景点的，因此我们的Loss function使用的仍然是之前所介绍的Focal Loss：

Lfocal(pt)=−αt(1−pt)γlog⁡(pt)L_{focal}(p_t) = -\alpha_t(1 - p_t)^\gamma \log(p_t)Lfocal​(pt​)=−αt​(1−pt​)γlog(pt​)

pt={pfor foreground point1−potherwise p_t=\left\{ \begin{array}{rcl} p & & {for\ foreground\ point}\\ 1-p & & {otherwise} \end{array} \right. pt​={p1−p​​for foreground pointotherwise​

其中，这个ppp也是经过softmaxsoftmaxsoftmax之后的结果，也就是：

  • p=pforeground=ec1ec0+ec1,pforeground=1−p=ec0ec0+ec1p = p_{foreground} = \frac{e^{c_1}}{e^{c_0} + e^{c_1}}, p_{foreground} = 1 - p = \frac{e^{c_0}}{e^{c_0} + e^{c_1}}p=pforeground​=ec0​+ec1​ec1​​,pforeground​=1−p=ec0​+ec1​ec0​​

4.2.2 回归

作为三维位置回归，所需要的参数主要有七个：x,y,z,l,w,h,θx, y, z, l, w, h, \thetax,y,z,l,w,h,θ

对于其中的x,z,θx, z, \thetax,z,θ，这里所使用的方法是Classification + Finetuning residual（文章中的高程轴为y轴）。对于其他的则是直接回归。因为深度学习中，分类的精度一定是大于回归的。

以xxx为例，首先将这个proposal周围的x轴和z轴区域分成好几个格子，看看中心点在哪个格子里面，然后再在这个格子里面进行小范围的回归。

4.3 ROI pooling

  • 每一个点都有一个proposal真的太多了，因此需要进行NMS，也就是按照置信度选出部分proposal进行操作。对于训练过程而言，一般选择300个，inference一般选择100个。
  • 然后对于每个proposal，把框放大，如上面整体流程图左下角的图所示。这样做是因为，要进行预测分类之类的操作，只关注这个物体可不行，也需要去关注它周围所处的环境。
  • 在这个enlarged box里面收集信息，所需要收集的信息包括：
    • 每个点的坐标xi,yi,zix_i, y_i, z_ixi​,yi​,zi​，坐标点会进行Canonical Transformation操作
    • 每一个点的反射率rir_iri​
    • 每一个点与原点的距离，di=xi2+yi2+zi2d_i = \sqrt{x_i^2 + y_i^2 + z_i^2}di​=xi2​+yi2​+zi2​​
    • 对于里面的点属于前景点还是背景点，mim_imi​
    • 将这些东西和第一步中每一个点的feature map进行一下concatenate，然后放进MLP里面。

其中提到了Canonical Transformation，这个操作从本质上来说，就是一种Normalization。简单来说就是每一个点都减去这个enlarged box的中心点坐标，然后让它旋转到x坐标轴方向。

5 Fusion

这里介绍一些二维图像和三维点云融合的模型：

5.1 Frustum PointNet

这个的思路也相当简单，就是先在图片上跑一个MaskRCNN，然后找到的proposal投影到点云中，只对投影到的点云进行操作。这样就大大减少了所需要处理的点云数据量。但是这个模型效果很不好，具体来说存在这些缺陷：

  • MaskRCNN并不是万无一失的，万一这个模型没有检测到，那就全都完蛋了。照片又很容易被遮挡，因此这是一个相当现实的问题；
  • 现在的相机有一些仍然是rolling式的，也就是每一个像素都不是同一时间的，点云更不用说，LiDAR获取的各个点也并不是同一时间的，因此也会存在一定的误差；
  • 相机和LiDAR也并不处于同一位置，因此会造成更大的误差。

5.2 PointPainting

这个就是先对图片进行Semantic Segmentation，然后将这些点投影到点云中去，进行点云数据的加强，然后就可以进行其他操作了。

同样存在上面所说的问题，但是由于这是一个弱结合，并没有信息的缺失，因此这是一个纯加强，效果也相对较好。
