发表于2023-08-04|更新于2023-08-11|点云

1 法向量

用曲面去拟合点云数据点后，某个点的法向量就是过这个点与该曲面相切的平面的法向量，如图所示。

1.1 法向量的应用

  • **对平面进行检测。 ** 如地面上的法向量应该都是竖直向上的，这种时候就可以通过法向量来判断地面范围。
  • 分割算法。 如下图所示的圆柱体，它侧面的曲面的法向量和两个底面的法向量是互相垂直的，因此可以通过这种方法来分离圆柱体的侧面和底面。
  • 还有诸如在深度学习相关环节中的应用等其他应用。

1.2 法向量的求法

我们思考一下，我们最终求出来协方差矩阵具有很多特征向量，然后第一个特征向量对应的就是最主要的成分，那最后一个特征向量是否就是最不主要的成分，也就是法向量呢？依据此，我们得到有以下步骤：

  • 选择我们要求的点；
  • 寻找这个点的邻域点以组成平面；
  • 对这个平面进行PCAPCAPCA；
  • 之后我们在其中寻找最不主要的向量，也就最后一个特征向量；
  • 我们也可以对曲率进行计算，也就是λ3=(λ1+λ2+λ3)\lambda_3 = (\lambda_1 + \lambda_2 + \lambda_3)λ3​=(λ1​+λ2​+λ3​)。

1.3 对噪声的处理

难免我们的数据中会存在上下左右的波动，这种情况下就会造成最终拟合出的曲面并不光滑。因此我们可以采用以下方法：

1.3.1 对邻域点的选择

这个就非常依赖于经验了。我们可以选择半径较大的邻域或者半径较小的邻域，这其中存在有各自的优缺点：

  • 大半径。 这会让曲面更加光滑，但是容易被代表其他物体的点所影响。
  • 小半径。 这会让曲面更加尖锐准确，但是很容易被噪声所影响。

1.3.2 使用其他数据

可以使用横坐标、纵坐标和高程以外的数据，比如说可能会有颜色数据，那么这种情况下就可以去选择与所选点颜色相近的点。比如说一辆红车的话，可能所选半径中有一些黑色的点，那可能就是道路点了，就不予考虑啦。

另外如果使用激光雷达，那么会有反射率数据，我们同样可以选择反射率相近的点来作为邻域点。

那么实际上，就是使用这些数据来对其进行加权。再具体一点，我们所要求的就是

2 滤波器

滤波的概念是十分宽泛的，如上述的噪声去除同样是一种滤波。其他常见的还有降采样滤波（简单来说就是要识别一个人可能只需要100个点，而我们雷达探测到了1000个点，这是没有必要的而且还会增大运算量，因此我们就减少其中的点来提高运算效率）和上采样滤波（和降采样滤波相对的，常见的就是双边滤波）等。

2.1 噪声去除滤波器

2.1.1 PCL半径滤波(Radius Outlier Removal)

这是一个十分简单且实用的方法。

如上图所示，对每一个点进行操作，选取一个半径rrr，在每个点的半径rrr范围内找点，如果点的数量低于我们的规定值，我们就认为它是噪声点，就进行去除。比如说我们规定值为222，那么上图中黄色的点和绿色的点是不符合要求的，我们就给他去掉。

2.1.2 Statistical Outlier Removal（不会翻译了）

实际上是半径滤波的升级版。首先规定好临近点数量，然后开始找临近点。找到临近点之后，计算一下这些点到选定点的距离。假设其符合高斯分布d∼N(μ,σ)d\sim N(\mu, \sigma)d∼N(μ,σ)

μ=1mk∑i=1m∑j=1kdij,σ=1mk∑i=1m∑j=1k(dij−μ)2\mu = \frac{1}{mk} \sum_{i=1}^m \sum_{j=1}^k d_{ij}, \sigma = \sqrt{\frac{1}{mk} \sum_{i=1}^m \sum_{j=1}^k (d_{ij} - \mu) ^ 2}μ=mk1​∑i=1m​∑j=1k​dij​,σ=mk1​∑i=1m​∑j=1k​(dij​−μ)2​

对于每个点来说，都计算一下到它的邻居们的平均距离，如果这个平均距离在高斯分布之外的话，就认为它是噪声点，就直接给去除。如下面：

∑j=1kdij>μ+3σ ∣∣ ∑j=1kdij<μ−3σ\sum_{j = 1}^kd_{ij} > \mu + 3\sigma \ || \ \sum_{j = 1}^kd_{ij} < \mu - 3\sigma∑j=1k​dij​>μ+3σ ∣∣ ∑j=1k​dij​<μ−3σ

那么就认为这个点是离群的，直接去除。当然了，式子里面的σ\sigmaσ的倍数是可以自定的。

2.2 降采样滤波

2.2.1 体素滤波器(Voxel Grid Downsampling)

通过上面这个图直接言简意赅说明了这个算法是怎么回事！我们以二维空间为例，我们可以把它分为好几块，比如左边这个就是分成了一块，右边就是分成了四块。分多了会很准确，但就是数据量会比较大，运算量也会大到不行。然后就每一块选一个点来代表这一块。简单来说，要用几个点就分成几块。

那么，每一块里面的点该怎么选呢？主要用两种方法：

  • 平均值法。 也就是网格中的所有点各个坐标轴求平均值。但这样的问题就是可能里面各个点有标签，比如有的点代表桌子，有的点代表人，这肯定是没有办法去求平均值的，因此还需要投票操作。这样的好处就是很准确，但是计算量较大，会比较慢。
  • 随机点法。 也就是网格中的所有点随机选一个作为代表，很快但是并不准确。

那么接下来简单介绍一下实现步骤：

  • 计算所有点各个维度的最值，也就是三维的话要求出xmin,xmax,ymin,ymax,zmin,zmaxx_{min}, x_{max}, y_{min}, y_{max}, z_{min}, z_{max}xmin​,xmax​,ymin​,ymax​,zmin​,zmax​

  • 选定一下voxel grid的边长rrr

  • 计算一下各个维度都有多少个voxel grid，也就是有多少个rrr。那么很明显就会有

    Dx=(xmax−xmin)/rD_x = (x_{max} - x_{min}) / rDx​=(xmax​−xmin​)/r

    Dy=(ymax−ymin)/rD_y = (y_{max} - y_{min}) / rDy​=(ymax​−ymin​)/r

    Dz=(zmax−zmin)/rD_z = (z_{max} - z_{min}) / rDz​=(zmax​−zmin​)/r

  • 然后计算一下每一个点从各个维度出发在哪个voxel grid里面：

    hx=⌊(x−xmin)/r⌋h_x = \lfloor (x - x_{min}) / r \rfloorhx​=⌊(x−xmin​)/r⌋

    hy=⌊(y−ymin)/r⌋h_y = \lfloor (y - y_{min}) / r \rfloorhy​=⌊(y−ymin​)/r⌋

    hz=⌊(z−zmin)/r⌋h_z = \lfloor (z - z_{min}) / r \rfloorhz​=⌊(z−zmin​)/r⌋

    h=hx+hy∗Dx+hz∗Dx∗Dyh = h_x + h_y * D_x + h_z * D_x * D_yh=hx​+hy​∗Dx​+hz​∗Dx​∗Dy​

  • 也就是说现在每一个点都有一个hhh属性来说明它在哪一个voxel grid里面了，接下来我们对其进行排序，排序出的结果应该类似于0, 0, 0, 3, 3, 3, 5, 5, 5, 5, …然后相同的里面选一个就可以了。

但是这也是存在一定的问题的，比如说……如果分成的空间很多，那会不会开数组就会占用内存过大？又或者，排序的时间复杂度为O(NlogN)O(NlogN)O(NlogN)，那么点很多的时候是不是所消耗的时间又太多了呢？

我们思考一下，实际上大部分空间是没有点的，也就是我们所设计的hhh个voxel grid中其实大部分是没有被用到的。比如说我们有10000个点，但是最终所用到的voxel grid只有95个。那实际上我们就可以设计有100个容器，最后最好的结果就是有五个容器是空的，95个容器是满的。这样只需要把这些点放入这些容器就可以了。那么我们所使用的方法就是哈希表。比如说，我们算出来每一个点的hhh了，然后使用哈希函数获得哈希值，比如我们可以使用：

hash(hx,hy,hz)=(hx+hy∗Dx+hz∗Dx∗Dy)%container_sizehash(h_x, h_y, h_z) = (h_x + h_y * D_x + h_z * D_x * D_y) \% container\_sizehash(hx​,hy​,hz​)=(hx​+hy​∗Dx​+hz​∗Dx​∗Dy​)%container_size

这样会有R3→RR^3 \rightarrow RR3→R，最终的结果一定会小于container_sizecontainer\_sizecontainer_size。但是又会有新的问题，那就是会导致重复，也就是冲突：

hash(hx,hy,hz)=hash(hx′,hy′,hz′),hx≠hx′ or hy≠hy′ or hz≠hz′hash(h_x, h_y, h_z) = hash(h_x', h_y', h_z'), h_x \neq h_x'\ or\ h_y \neq h_y'\ or\ h_z \neq h_z'hash(hx​,hy​,hz​)=hash(hx′​,hy′​,hz′​),hx​=hx′​ or hy​=hy′​ or hz​=hz′​

那么这个时候，我们就这个容器里面的点移到其他容器中，然后再把最新的点放进去。

2.2.2 Farthest Point Sampling(FPS)

**这种方法经常在深度学习中有所应用。**其步骤大概为：

  • 首先随机选中一个点，作为第一个FPSFPSFPS的点
  • 之后重复下面步骤，直到已经找寻到我们所规定数量的点：
    • 首先，在原点云中计算和最近的FPSFPSFPS点的距离
    • 然后从中选出距离最大的点作为最新的FPSFPSFPS点

2.3 上采样滤波

常用的方法就是Bilateral Filter。在那之前，我们应该了解一下目前常用的高斯模糊是什么原理。

高斯模糊实际上就是对图片的每一个像素进行分析，将这个像素的颜色变为其周边像素颜色的加权平均值。至于怎么加权，自然是使用高斯分布（正态分布）函数进行加权，距离更近的点权重更高，公式为：

GB[I]p=Σq∈SGσ(∣∣p−q∣∣)IqGB[I]_p = \Sigma_{q\in S}G_{\sigma}(||p - q||)I_qGB[I]p​=Σq∈S​Gσ​(∣∣p−q∣∣)Iq​

Gσ(x)=12πσ2exp(−x22σ2)G_\sigma(x) = \frac{1}{2\pi\sigma^2}exp(-\frac{x^2}{2\sigma^2})Gσ​(x)=2πσ21​exp(−2σ2x2​)
