SUMO（零）—— 地图文件转化

发表于2022-05-25|更新于2022-06-06|sumo

对于SUMO来说，很多时候都是需要从其他文件转化成SUMO的 .net.xml.net.xml.net.xml 文件的。这篇文章我主要记载的是我如何从 .shp.shp.shp 文件转化成 .net.xml.net.xml.net.xml 文件的。

SUMO的官方文档说，它自带的 netconvertnetconvertnetconvert 是可以转化 .shp.shp.shp 文件的。但是那个转化在正式版的SUMO中是不支持的，只能下载不稳定的版本才能用。我切换系统在 Windows 下使用后，发现还是很不好用，还要根据地图属性来添加各种参数。于是我就放弃了这个做法。

最后还是决定采用将 .shp.shp.shp 转化为 .osm.osm.osm 然后再转化为 net.xmlnet.xmlnet.xml 的方法。

本文倒序，先转化为 .net.xml.net.xml.net.xml，再讲怎么转化为 .osm.osm.osm。

1 .osm转化为.net.xml

将转换之后的直接导入，发现结果为：

No nodes loaded.

首先，要解释一下 .osm.osm.osm 文件的格式。这个文件本质上还是xml文件。主要包括 node 和 way 两部分，分别代表节点和道路。

就是类似于这样的。

那么为什么 No nodes loaded 呢？因为能够转化必须要满足以下两个条件：

  • 要有node和way两种tag的对象。（这个是满足的）
  • 要有部分way里面有 k="highway"的标签（不满足）

不是这样的。这个其实我感觉，每一个shp文件他都是不一样的。我对着Open Street Map的文件一个一个进行比对，最后终于发现，道路的类型是存储在了 kindkindkind 里面。这属于是加密通话了，什么0601、0402、0814的，什么玩意儿！

然后，我一条边一条边进行比对，记录总结之后发现了对应规律：

kind 对应值
0601/080a 三级公路
0602 二级公路
0402/0412/080a 一级公路
0801 living_street
0202/060a 快速路
0814 居民路

然后进行转换，发现转化出来的有些边居然是断裂的！赶紧对应看一下，发现还有一个key：kind_numkind\_numkind_num。有的边对应好几种类型！（其实最多两种）

有两种类型，就比如“0601|080a”这样子，对着OpenStreetMap的文件进行对比，发现这两个里面至少有一个是上面表格中列出来的，并且跟列出来的那一个或者两个所对应的kind是一样的！也就是说遇到这种情况，就把两边都拆出来，第一个在表格里面就按照第一个的kind处理，否则就按照第二个来处理。

2 .shp转化为.osm

在前面 no nodes loaded 之前，肯定还会有些别的错误，会告诉你某一个属性是不合法的。这个时候，你就需要先去ArcMap先把那一个属性删掉，然后保存下来。

之后要进行转码。因为原本文件大概率是不支持中文字符的，容易出现乱码。这个时候，就可以去网上找GIS转码器来进行使用。可以看看这个QGIS转码。

2.1 JOSM转化

首先下载JOSMJOSMJOSM，JOSM官网。但由于我不是WindowsWindowsWindows用户，所以不是很了解在WindowsWindowsWindows下如何操作。macOSmacOSmacOS可以直接利用homebrewhomebrewhomebrew来进行安装。

安装完成后打开，在菜单栏找到“影像”

然后点击“影像首选项”。

点击那个插件按钮，然后搜索opendataopendataopendata，下载安装即可。这样就可以把shapeshapeshape文件拖进去然后就可以另存为osmosmosm格式了。
