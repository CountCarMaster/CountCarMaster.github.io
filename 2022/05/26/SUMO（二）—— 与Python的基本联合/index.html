这篇文章主要 $SUMO$ 与 $Python$ 的基本联合写法，还不涉及 $TraCI$ 接口。本文举的例子的目的是探索一个路口中到达率大小与停车率的关系

0 基础准备

0.1 知识准备

在看这篇文章之前，你需要：

  • 掌握基本的 $Python$ 知识（链接是我的好兄弟 川$shit$ 整理的相关知识）；
  • 掌握基本的 $NumPy$ 知识；
  • 掌握基本的 $xml.etree.ElementTree$ （或者其他处理 $xml$ 文件的方法）知识；
  • 掌握基本的 $Matplotlib$ 知识；
  • 掌握基本的 $SUMO$ 相关文件 的知识。

  • 一个路网文件（$.net.xml$)

  • 后面则是对仿真文件进行转换。由于我们会调用好多次生成路由文件的函数，而每次调用的 $period$ 都不一样，我们要保留每种情况的仿真结果。于是我们让每次仿真的仿真结果保存为 $cross.output + str(period) + .xml$ 形式。比如说我们某一次仿真的 $period$ 为 $0.05$ ，那么我们保存的输出结果就会是 $cross.output0.05.xml$ 。这样就不会出现名字重复的问题，我们也可以找到各个 $period$ 所对应的仿真结果。如果这一部分代码看不明白请参照 0.1 中的 “掌握基本的 $xml.etree.ElementTree$ （或者其他处理 $xml$ 文件的方法）知识；”
  • 最后则是运行 $SUMO$ 进行仿真，每 $100$ 秒保存一次结果。

因为我们之前已经提到，仿真时间为 $1000$ 秒，而我们又是每 $100$ 秒进行一次数据的获取，所以就有 $10$ 组结果。

基本结构很简单，就是 $root$ 下面有 $10$ 个 $tag$ 为 $timestep$ 子节点，也就是记录了每 $100$ 秒的情况。 $timestep$ 下面则是 $tag$ 为 $vehicle$ 子节点，记录了在这个时间点的每一辆车的具体数据，包括 $id$ ，坐标，速度，车辆类型，所在车道，所在车道的位置等信息。不同的应用会用到不同的信息，而我们所用到的信息就是速度，也就是 $speed$ 这个 $attribute$ 。
